# Evaluation

## 평가 목적

이 문서는 Course Study RAG Tutor v1의 검색 결과를 수동으로 평가하기 위해 작성되었습니다.

v1은 키워드 기반 검색을 사용하므로, 질문과 문서 chunk 사이의 단어 겹침이 많을수록 높은 점수를 얻습니다. 따라서 정확한 키워드가 포함된 질문에서는 잘 동작하지만, 표현이 애매하거나 동의어가 사용된 질문에서는 검색 품질이 떨어질 수 있습니다.

## 평가 기준

| 기준 | 설명 |
|---|---|
| Success | 질문 의도에 맞는 chunk가 Top-3 안에 포함됨 |
| Partial | 관련 chunk가 일부 포함되지만 불필요한 chunk도 함께 검색됨 |
| Fail | 질문 의도와 맞지 않는 chunk가 검색됨 |

## v1 분석

v1은 질문에 명확한 키워드가 포함되어 있을 때 안정적으로 관련 chunk를 검색했습니다. 예를 들어 “기본 키와 외래 키의 차이”와 같이 문서에 포함된 핵심 용어가 질문에 그대로 들어간 경우, 관련 chunk가 Top-3 안에 포함되었습니다.

하지만 “키는 무엇인가요?” 또는 “학습은 무엇인가요?”처럼 질문이 짧거나 모호한 경우에는 여러 문서의 chunk가 함께 검색될 수 있습니다. 이는 키워드 기반 검색이 문맥적 의미보다는 단어 일치 여부에 크게 의존하기 때문입니다.

## v2 개선 방향

v2에서는 sentence-transformers 기반 임베딩 검색을 적용하여 질문과 chunk를 벡터로 변환하고, cosine similarity를 기준으로 의미적으로 가까운 chunk를 검색할 예정입니다.

이를 통해 다음 문제를 개선할 수 있습니다.

- 정확한 단어가 일치하지 않아도 의미가 비슷한 chunk 검색
- 짧은 질문에 대한 검색 품질 개선
- 키워드 중심 검색에서 의미 중심 검색으로 확장

## v2.1 Hybrid Retrieval Evaluation

v2에서 sentence-transformers 기반 임베딩 검색을 적용했지만, 일부 모호한 질문에서는 의도와 다른 chunk가 상위에 노출되는 문제가 있었습니다.

예를 들어 “데이터를 구분하는 식별자는 무엇인가요?”라는 질문에서 초기 v2는 군집화 chunk를 상위에 노출했습니다. 이는 질문에 포함된 “데이터”, “구분”이라는 표현이 군집화 설명과 의미적으로 가깝게 계산되었기 때문입니다.

이를 개선하기 위해 v2.1에서는 다음 세 가지 점수를 함께 사용했습니다.

- embedding similarity score
- keyword overlap score
- domain-specific boost score

## 분석

v2.1에서는 임베딩 검색의 의미 기반 검색 장점을 유지하면서, 특정 개념 질문에 대해서는 도메인 보정 점수를 적용해 더 적절한 chunk가 상위에 오도록 개선했습니다.

다만 현재 보정 규칙은 샘플 문서에 맞춘 단순 규칙이므로, 대규모 문서나 다양한 과목으로 확장할 경우에는 BM25, 형태소 분석기, reranker 모델 등을 추가하는 방식으로 개선할 수 있습니다.
